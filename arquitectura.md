# Arquitectura del sistema

Este sistema en la nube está diseñado para ofrecer alta disponibilidad y organización por capas.

## Componentes principales
- Cliente web
- Servidor de aplicaciones
- Base de datos
- Balanceador de carga

## Objetivo
Permitir que los usuarios accedan al sistema de forma estable y segura.

## Servicios en la nube
- Cómputo para la lógica del sistema
- Almacenamiento para archivos
- Base de datos administrada
- Monitoreo para disponibilidad

## Beneficios
- Escalabilidad
- Resiliencia
- Mejor control de recursos

## Seguridad
- Acceso con usuario y contraseña
- Cifrado de datos en tránsito
- Cifrado de datos en reposo
- Control de acceso por roles
- Respaldo semanal

## Protección adicional
- Uso de autenticación multifactor
- Registro de eventos de acceso
- Políticas de contraseñas seguras

## Monitoreo de seguridad
- Alertas por actividad sospechosa
- Revisión de logs
- Auditoría de cambios

## Operacion
- Monitoreo basico del sistema
- Revision manual de incidentes
